Securing MCP Servers: Tool Access Control for AI Agents

Manjunath K G April 2026 10 min read

Model Context Protocol (MCP) turns LLMs into systems that act on the world — read S3 buckets, pull Jira tickets, scan GitHub repos. Building an AWS MCP server with 70+ tools for GRC evidence collection taught me that MCP is a new attack surface, not just integration glue.

The Trust Problem

An MCP server exposes capabilities. An agent decides when to invoke them. A malicious or confused agent can exfiltrate data, modify resources, or chain tools in unintended ways. The server must assume the caller is untrusted.

Security Principles for MCP Servers

1. Read-Only by Default

GRC evidence collection rarely needs write access. Expose list/describe/get operations. Require separate elevated scopes — with human approval — for mutating tools.

2. Credential Isolation

MCP servers should use short-lived, scoped credentials per tenant. Never share a root AWS key across clients. Map agent sessions to IAM roles with minimal permissions.

3. Parameter Validation

def list_buckets(region: str, prefix: str | None = None):
    if region not in ALLOWED_REGIONS:
        raise PermissionError("Region not in scope")
    if prefix and ".." in prefix:
        raise ValueError("Invalid prefix")

4. Audit Every Invocation

Log tool name, tenant ID, parameters (redacted), timestamp, and calling agent ID. Feed into your GRC audit trail — clients will ask.

5. Rate Limiting & Quotas

Prevent runaway agents from hammering cloud APIs or racking up costs. Per-session and per-tenant limits.

Multi-Cloud MCP Architecture

We extended evidence collection across AWS, Azure, GCP, Jira, and GitHub via dedicated MCP agents. Each domain gets its own server process with isolated credentials. A composite orchestrator agent coordinates them but cannot bypass per-server policies.

Common mistake: One mega-MCP server with every tool and one admin credential. Split by domain and privilege level.

Testing MCP Security

  • Fuzz tool parameters with injection payloads
  • Attempt cross-tenant access with swapped session tokens
  • Red-team agent prompts that chain benign tools into harmful sequences
  • Verify credential rotation and revocation paths

MCP is how agents touch production systems. Secure it like an API gateway — because that's what it is.
