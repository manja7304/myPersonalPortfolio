Temporal Workflows for Reliable AI Agent Pipelines

Manjunath K GMay 20269 min read

LLM calls are flaky. APIs timeout. Agents loop. If your multi-agent pipeline lives in a single Python script, one failure means restarting a 3-hour job. Temporal turns agent orchestration into durable, resumable workflows.

Why Not Cron + Celery?

Celery handles tasks, not long-running stateful workflows. Temporal gives you:

  • Durable execution — workflow state survives process crashes
  • Automatic retries — with backoff policies per activity
  • Visibility — inspect running workflows in the UI
  • Saga patterns — compensate on partial failure

Mapping Agents to Activities

@workflow.defn
class PentestWorkflow:
    @workflow.run
    async def run(self, engagement_id: str):
        scope = await workflow.execute_activity(
            classify_scope, engagement_id, start_to_close_timeout=timedelta(minutes=5))
        wave1 = await asyncio.gather(*[
            workflow.execute_activity(run_agent, agent, scope)

Checkpointing Between Waves

Run discovery agents in wave 1, exploitation agents in wave 2 — each wave checkpoints to the evidence store before the next begins. If wave 2 fails at agent 15, resume from agent 15, not from zero.

Production Lessons

  • Set start_to_close_timeout per agent type — web scans take longer than classifiers
  • Use heartbeats for long LLM calls so Temporal knows the activity is alive
  • Version workflows carefully — use patching for zero-downtime updates
  • Keep activities idempotent — retries will happen
