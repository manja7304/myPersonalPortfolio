Hybrid Search for Agentic RAG: Vectors + Keywords

Manjunath K GMay 20268 min read

Pure vector search misses exact matches — CVE IDs, error codes, API endpoint paths. Pure keyword search misses semantic similarity. Production agentic RAG needs both.

The Hybrid Pipeline

  1. Query analysis — agent decides if query is semantic, exact-match, or hybrid
  2. Parallel retrieval — embedding search (Pinecone/pgvector) + BM25 keyword index
  3. Reciprocal Rank Fusion — merge results without tuning arbitrary weights
  4. Reranking — cross-encoder or LLM reranker on top-k candidates
  5. Context assembly — dedupe, truncate, tag as untrusted

When Agents Choose Retrieval Strategy

In agentic RAG, the agent doesn't just answer — it decides what to search and how. A security agent querying "CVE-2024-1234" should route to keyword search. A question about "authentication best practices" routes to semantic search.

Chunking for Security Docs

Security documentation has tables, code blocks, and config snippets. Chunk by logical sections, not fixed token windows. Preserve headers as metadata for filtered retrieval ("only search OWASP guides").

Hybrid search improved our RAGAS context precision scores significantly on technical queries with exact identifiers — the kind security workflows produce constantly.

Storage Options

  • pgvector + PostgreSQL full-text — single DB, good for moderate scale
  • Pinecone + Elasticsearch — separate specialized indexes at scale
  • Chroma — rapid prototyping with built-in hybrid support
