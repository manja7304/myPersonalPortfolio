RAG Evaluation with RAGAS: Stop Shipping Broken Retrieval

Manjunath K G May 2026 9 min read

RAG failures are silent. The model responds confidently with wrong information, citing documents that don't support the claim. Users trust it because it sounds authoritative. RAGAS gives you numbers before your customers find the bugs.

Core Metrics

  • Faithfulness — Is the answer grounded in retrieved context, or hallucinated?
  • Answer Relevancy — Does the response actually address the question?
  • Context Precision — Are retrieved chunks relevant to the query?
  • Context Recall — Did retrieval find the information needed to answer?

Building an Evaluation Pipeline

# Simplified evaluation flow
test_set = load_golden_qa_pairs()  # question + expected sources
for case in test_set:
    chunks = retriever.search(case.question)
    answer = generator(chunks, case.question)
    scores = ragas.evaluate(case, chunks, answer)
    log_to_dashboard(scores)

When to Run Evaluations

  1. Before deploying prompt changes
  2. After embedding model or vector store updates
  3. When switching LLM providers or model tiers
  4. Weekly on a held-out regression set

Regression Gates in CI

Set thresholds: faithfulness ≥ 0.85, relevancy ≥ 0.80. Block merges that drop metrics below baseline. We improved both by ~30% through iterative prompt tuning — but only because we measured before and after.

Pro tip: Build golden datasets from real user queries (anonymized) and known failure cases. Synthetic-only eval sets miss production edge cases.

Beyond RAGAS

Combine automated metrics with human review for high-stakes outputs (security findings, compliance reports). RAGAS catches retrieval drift; humans catch domain nonsense that scores well but misleads experts.

Agentic RAG Considerations

When agents decide what to retrieve and when, evaluate the full trajectory — not just final answers. Did the agent retrieve before answering? Did it over-retrieve sensitive context? Multi-step agent eval is harder but essential for production agentic RAG.
