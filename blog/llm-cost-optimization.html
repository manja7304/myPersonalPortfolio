LLM Ops Cost Engineering

Cutting LLM Token Costs by 30% Without Sacrificing Quality

Manjunath K G April 2026 8 min read

At scale, LLM costs aren't a line item — they're a product constraint. Running dozens of agents per client engagement, we needed to cut token spend without degrading report quality. Here's what moved the needle.

1. Model-Tier Routing with LiteLLM

Not every step needs GPT-4-class reasoning. Route by task complexity:

  • Tier 1 (fast/cheap) — classification, routing, format validation
  • Tier 2 (balanced) — analysis, summarization, standard reports
  • Tier 3 (capable) — complex reasoning, novel vulnerability interpretation

LiteLLM centralizes routing, fallbacks, and cost tracking across providers.

2. Prompt Scaffolding Over Prompt Bloat

Replace verbose instructions with structured scaffolds: few-shot examples, chain-of-thought templates, and output schemas. Shorter prompts + better outputs. We templatized system prompts per agent type and version-controlled them.

3. Response Caching

Identical or near-identical queries (common in batch assessments) hit cache instead of the API. Hash prompts + context for cache keys. Invalidate on prompt version bumps.

4. Context Window Discipline

Agents over-fetch context. Implement retrieval budgets: max chunks, max tokens per source, summarization of long documents before injection. Every token in context is a token billed twice (input + influence on output).

5. Evaluation-Gated Changes

Never optimize cost without measuring quality impact. RAGAS scores and human spot-checks on report samples confirmed our 30% token reduction didn't hurt faithfulness or relevancy.

Report generation went from 6 hours to 10 minutes (~97% time reduction) partly because we stopped sending entire evidence dumps to the model — structured summaries instead.

Metrics to Track

  • Cost per engagement / per agent type
  • Input vs. output token ratio
  • Cache hit rate
  • Quality scores pre/post optimization

Cost optimization is an engineering discipline. Treat it like performance tuning — measure, change, verify, repeat.
